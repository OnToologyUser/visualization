Territorio: Description back to ToC

La siguiente figura muestra las principales clases definidas en este vocabulario (todas ellas utilizando el prefijo esadm), así como su relación con la taxonomía de clases de las ontologías de GeoNames, schema.org y DBpedia.

Y en la siguiente figura se muestran las propiedades definidas en este vocabulario, tal y como se ha descrito anteriormente. Asimismo hemos incluido en la imagen algunas de las propiedades que son relevantes para las clases que son subclase de la clase Feature de GeoNames, como featureClass, featureCode, name, officialName, etc. Concretamente, en el caso de featureCode, hemos especificado los códigos con los que se relacionan de la ontología de GeoNames. Y para no hacer la lectura de la imagen muy complicada, se ha omitido el valor de la propiedad featureClass, aunque en los ejemplos de la sección 3.1 se incluyen. En estos casos, la featureClass se relaciona con los recursos geonames:A para las regiones, y geonames:P para las entidades que se incluyen dentro de los municipios.

También se pueden utilizar, y se recomienda utilizar, propiedades generales de Dublin Core (por ejemplo, dct:title y dct:identifier) y otras propiedades generales como rdfs:label, rdfs:comment, owl:sameAs, etc. Finalmente, no se muestra la propiedad esadm:estado, que permite identificar el tipo de estado de un recurso, de acuerdo con un esquema de conceptos SKOS que identifica estados vigente, histórico y provisional, tal y como se propone en el modelo de direcciones de la Administración General del Estado v.2.

3.1 Ejemplos de uso back to ToC

A continuación mostramos algunos ejemplos del uso de algunas de estas clases y propiedades, con ejemplos procedentes de los portales de datos abiertos de Zaragoza y OpenDataAragón. Comenzaremos por las autonomías, e iremos bajando poco a poco hasta el nivel de la sección censal. Utilizaremos en todos estos ejemplos el formato Turtle

Veamos cómo podemos representar la autonomía de Aragón. Como se puede ver, declaramos que es una instancia de nuestra clase esadm:ComunidadAutonoma, que es equivalente al recurso correspondiente en http://datos.gob.es/, que tiene como geonames:featureCode una referencia al recurso geonames:A.ADMD, como se presentaba en una de las imágenes previas, y como geonames:featureClass una referencia al recurso geonames:A, y que tiene una serie de datos que parecen "repetidos", como su nombre común y nombre oficial. Esta "repetición" de los mismos valores con distintas propiedades es algo que permite facilitar el acceso a nuestros datos a aplicaciones que utilizan unos u otros vocabularios. Finalmente, el estado de este recurso es vigente, en el sentido de que la comunidad autónoma existe en la actualidad. Otros posibles estados son histórico y provisional.

        <http://opendata.aragon.es/recurso/sector-publico/territorio/Autonomia/Aragon>
        a                      esadm:ComunidadAutonoma ;
        owl:sameAs             <http://datos.gob.es/recurso/sector-publico/territorio/Autonomia/Aragon> ;
        owl:sameAs             <http://eurostat.linked-statistics.org/dic/geo#ES24>
        geonames:featureCode   <http://www.geonames.org/ontology#A.ADMD> ;
        geonames:featureClass  <http://www.geonames.org/ontology#A> ;
        esadm:estado           <http://vocab.linkeddata.es/datosabiertos/kos/sector-publico/territorio/tipoEstado/vigente> ;
        dcterms:identifier	   "XXXX" ; #TODO: no sé cuál se puede usar para una autonomía
        geonames:officialName  "Aragón" ;
        dcterms:title          "Aragón" ;
        geonames:name          "Comunidad Autónoma de Aragón" ;
        rdfs:label             "Comunidad Autónoma de Aragón" ;
        esadm:pais             <http://datos.gob.es/recurso/sector-publico/territorio/Pais/España> .

A continuación proporcionamos el detalle correspondiente a la provincia de Zaragoza. Podemos ver que el dcterms:identifier es 50, que es el que corresponde a esta provincia según el INE.

        <http://opendata.aragon.es/recurso/sector-publico/territorio/Provincia/Zaragoza>
        a                      esadm:Provincia ;
        owl:sameAs             <http://datos.gob.es/recurso/sector-publico/territorio/Provincia/Zaragoza> ;
        owl:sameAs             <http://eurostat.linked-statistics.org/dic/geo#ES243>
        geonames:featureCode   <http://www.geonames.org/ontology#A.ADMD> ;
        geonames:featureClass  <http://www.geonames.org/ontology#A> ;
        esadm:estado           <http://vocab.linkeddata.es/datosabiertos/kos/sector-publico/territorio/tipoEstado/vigente> ;
        dcterms:identifier	   "50" ;
        geonames:officialName  "Zaragoza" ;
        dcterms:title          "Zaragoza" ;
        geonames:name          "Provincia de Zaragoza" ;
        rdfs:label             "Provincia de Zaragoza" ;
        esadm:autonomia        <http://datos.gob.es/recurso/sector-publico/territorio/Autonomia/Aragon> ;
        esadm:pais             <http://datos.gob.es/recurso/sector-publico/territorio/Pais/España> .

Bajamos a la comarca en la que se sitúa la ciudad de Zaragoza, que se denomina D.C. Zaragoza. En este caso ya no añadimos el owl:sameAs a datos de datos.gob.es, puesto que no se llega a este nivel de detalle. 

        <http://opendata.aragon.es/recurso/sector-publico/territorio/Comarca/D.C._Zaragoza>
        a                      esadm:Comarca ;
        geonames:featureCode   <http://www.geonames.org/ontology#A.ADMD> ;
        geonames:featureClass  <http://www.geonames.org/ontology#A> ;
        esadm:estado           <http://vocab.linkeddata.es/datosabiertos/kos/sector-publico/territorio/tipoEstado/vigente> ;
        dcterms:identifier	   "XXXX" ; #TODO: no sé cuál se puede usar para una comarca
        geonames:officialName  "D.C. Zaragoza" ;
        dcterms:title          "D.C. Zaragoza" ;
        geonames:name          "Delimitación Comarcal de Zaragoza" ;
        rdfs:label             "Delimitación Comarcal de Zaragoza" ;
        esadm:provincia        <http://opendata.aragon.es/recurso/sector-publico/territorio/Provincia/Zaragoza> ;
        esadm:autonomia        <http://datos.gob.es/recurso/sector-publico/territorio/Autonomia/Aragon> ;
        esadm:pais             <http://datos.gob.es/recurso/sector-publico/territorio/Pais/España> .

Seguimos llegando a mayor nivel de detalle y nos centramos ahora en municipios, y concretamente en el municipio de Zaragoza. De nuevo, seguimos utilizando datos e identificadores procedentes del portal de datos abiertos de Aragón. En este caso decimos explícitamente que Zaragoza es una esadm:Ciudad, algo que es redundante, puesto que en nuestro vocabulario ya hemos definido que esadm:Ciudad es una subclase de esadm:Municipio. Esto puede ser útil, de nuevo, para facilitar a los sistemas que utilicen SPARQL encontrar esta información, sobre todo en el caso de que no tengan activada la posibilidad de realizar inferencias.

También se puede observar que, como en el caso anterior, añadimos muchas propiedades que se pueden inferir fácilmente al considerar la propiedad geosparql:sfWithin como una propiedad transitiva (por ejemplo, si decimos que Zaragoza pertenece a la comarca D.C. Zaragoza, y esta comarca pertenece a la provincia de Zaragoza, y esta provincia a la comunidad autónoma de Aragón, podemos inferir que la ciudad de Zaragoza pertenece a todos estos territorios simplemente mediante la especificación de que pertenece a la comarca). Sin embargo, esto simplifica el tipo de consultas SPARQL que podemos ejecutar sobre estos datos.

Finalmente, podemos ver que ahora el valor de la propiedad featureCode de GeoNames es geonames:P.PPL

        <http://opendata.aragon.es/recurso/sector-publico/territorio/Municipio/Zaragoza>
        a                             esadm:Municipio, esadm:Ciudad ;
        geonames:featureCode          <http://www.geonames.org/ontology#P.PPL> ;
        geonames:featureClass         <http://www.geonames.org/ontology#P> ;
        esadm:estado                  <http://vocab.linkeddata.es/datosabiertos/kos/sector-publico/territorio/tipoEstado/vigente> ;
        dcterms:identifier	          "50297" ;
        esadm:codigoMunicipioINE      "50297" ;
        esadm:codigoMunicipioCatastro "50900" ;
        geonames:officialName         "Zaragoza" ;
        dcterms:title                 "Zaragoza" ;
        geonames:name                 "Zaragoza" ;
        rdfs:label                    "Zaragoza" ;
        esadm:comarca                 <http://opendata.aragon.es/recurso/sector-publico/territorio/Comarca/D.C._Zaragoza> ;
        esadm:provincia               <http://opendata.aragon.es/recurso/sector-publico/territorio/Provincia/Zaragoza> ;
        esadm:autonomia               <http://datos.gob.es/recurso/sector-publico/territorio/Autonomia/Aragon> ;
        esadm:pais                    <http://datos.gob.es/recurso/sector-publico/territorio/Pais/España> .

A partir de este momento pasamos a considerar datos que pertenecen y son ofrecidos por el portal de datos abiertos de la ciudad de Zaragoza. Concretamente veremos cómo podemos utilizar este vocabulario para representar información de distritos, barrios, juntas administrativas y secciones censales.

Comenzamos incluyendo un ejemplo de un distrito, concretamente el distrito 12 de Zaragoza. Se puede observar que la URI de este distrito comienza ahora por www.zaragoza.es/api/, que es la base de la propuesta de datos abiertos del ayuntamiento de Zaragoza.

        <http://www.zaragoza.es/api/recurso/sector-publico/territorio/Distrito/12>
        a                      esadm:Distrito ;
        geonames:featureCode   <http://www.geonames.org/ontology#P.PPLX< ;
        geonames:featureClass  <http://www.geonames.org/ontology#P> ;
        esadm:estado           <http://vocab.linkeddata.es/datosabiertos/kos/sector-publico/territorio/tipoEstado/vigente> ;
        dcterms:identifier     "12" ;
        geonames:officialName  "Barrios Rurales Oeste" ;
        dcterms:title          "Barrios Rurales Oeste" ;
        geonames:name          "Distrito Barrios Rurales Oeste" ;
        rdfs:label             "Distrito Barrios Rurales Oeste" ;
        esadm:municipio        <http://opendata.aragon.es/recurso/sector-publico/territorio/Municipio/Zaragoza> ;
        esadm:comarca          <http://opendata.aragon.es/recurso/sector-publico/territorio/Comarca/D.C._Zaragoza> ;
        esadm:provincia        <http://opendata.aragon.es/recurso/sector-publico/territorio/Provincia/Zaragoza> ;
        esadm:autonomia        <http://datos.gob.es/recurso/sector-publico/territorio/Autonomia/Aragon> ;
        esadm:pais             <http://datos.gob.es/recurso/sector-publico/territorio/Pais/España> .

Aunque anteriormente hemos añadido información de comarca, provincia, autonomía y país, realmente no es habitual que se incluya este tipo de información cuando nos encontramos ya en un nivel territorial por debajo del municipio, y la recomendación principal sería la de sólo hacer referencia al municipio, mediante esadm:municipio.

A continuación pasamos a describir un barrio de Zaragoza, concretamente el barrio Villarrapa. En el caso de que un barrio esté contenido dentro de un distrito (algo que no ocurre siempre), añadiríamos una tripleta más esadm:distrito <<URIMunicipio>>

        <http://www.zaragoza.es/api/recurso/sector-publico/territorio/Barrio/32>
        a                      esadm:Barrio ;
        geonames:featureCode   <http://www.geonames.org/ontology#P.PPLX> ;
        geonames:featureClass  <http://www.geonames.org/ontology#P> ;
        esadm:estado           <http://vocab.linkeddata.es/datosabiertos/kos/sector-publico/territorio/tipoEstado/vigente> ;
        dcterms:identifier	   "32" ;
        geonames:officialName  "Villarrapa" ;
        dcterms:title          "Villarrapa" ;
        geonames:name          "Barrio Villarrapa" ;
        rdfs:label             "Barrio Villarrapa" ;
        esadm:municipio        <http://opendata.aragon.es/recurso/sector-publico/territorio/Municipio/Zaragoza> .

Y terminaremos con la junta de vecinos Montañana. En este caso todos los tipos de nombre coinciden, aunque no tiene por qué ser el caso. Como en el caso anterior, si una junta está dentro de un distrito (algo que no ocurre siempre), añadiríamos una tripleta más esadm:distrito <<URIMunicipio>>, y algo similar se haría si alguna pertenece a algún barrio, con esadm:barrio.

        <http://www.zaragoza.es/api/recurso/sector-publico/territorio/JuntaAdministrativa/20>
        a                      esadm:JuntaAdministrativa ;
        geonames:featureCode   <http://www.geonames.org/ontology#P.PPLX> ;
        geonames:featureClass  <http://www.geonames.org/ontology#P> ;
        esadm:estado           <http://vocab.linkeddata.es/datosabiertos/kos/sector-publico/territorio/tipoEstado/vigente> ;
        dcterms:identifier     "20" ;
        geonames:officialName  "Junta de Vecinos MONTAÑANA" ;
        dc:title               "Junta de Vecinos MONTAÑANA" ;
        geonames:name          "Junta de Vecinos MONTAÑANA" ;
        rdfs:label             "Junta de Vecinos MONTAÑANA" ;
        esadm:municipio        <http://opendata.aragon.es/recurso/sector-publico/territorio/Municipio/Zaragoza> .

Finalmente, vamos a ver cómo se representaría una sección censal. En todos los casos, las secciones censales se consideran para todos los efectos dentro de un distrito, por lo que su URI puede contener la referencia al Distrito también, para que sea una URI cool. 

        <http://www.zaragoza.es/api/recurso/sector-publico/territorio/Distrito/12/SeccionCensal/001>
        a                      esadm:SeccionCensal ;
        geonames:featureCode   <http://www.geonames.org/ontology#P.PPLX> ;
        geonames:featureClass  <http://www.geonames.org/ontology#P> ;
        esadm:estado           <http://vocab.linkeddata.es/datosabiertos/kos/sector-publico/territorio/tipoEstado/vigente> ;
        dcterms:identifier     "50297312001" ;
        geonames:officialName  "Distrito 12 - Sección censal 001" ;
        dc:title               "Distrito 12 - Sección censal 001" ;
        geonames:name          "Distrito 12 - Sección censal 001" ;
        rdfs:label             "Distrito 12 - Sección censal 001" ;
        esadm:distrito         <http://www.zaragoza.es/api/recurso/sector-publico/territorio/Distrito/12> .
        esadm:municipio        <http://opendata.aragon.es/recurso/sector-publico/territorio/Municipio/Zaragoza> .

Con esto hemos proporcionado un buen número de ejemplos sobre unidades territoriales, aunque también hemos dejado fuera algunas de las clases y propiedades definidas en este vocabulario, como por ejemplo esadm:CiudadAutonoma, que sería aplicable a Ceuta y Melilla, o algunos otros conceptos como esadm:EntidadColectiva, esadm:EntidadSingular, esadm:NucleoPoblacion y esadm:Diseminado.

Asimismo, para cualquiera de estos recursos podemos representar, si disponemos de esta información, su geometría, para lo cual se recomienda también el uso de GeoSPARQL.         geosparql:hasGeometry         <http://opendata.aragon.es/recurso/sector-publico/territorio/Municipio/Zaragoza/geometria> .
